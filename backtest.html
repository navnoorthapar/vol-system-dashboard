C16 Methodology Audit — Five Flaws Found, Three Fixed, Two Quantified
Self-audit of the backtest methodology (2026-06-11). All numbers on this page are produced under the corrected pipeline.
# Flaw Resolution Evidence
1 PDV look-ahead bias FIXED — PDVLinear retrained per backtest year on strictly pre-year returns (walk-forward), injected into the pdv_iv_spread feature. S1/S1C now trade a true causal PDV forecast
2 Circular classifier feature FIXED — vvix removed from training features (the R2 label is defined as VVIX>threshold, so the old 86.2% accuracy partly measured rule-recovery). Classifier must now predict R2 from market structure
3 Retrospective Kelly netting FIXED — correlation/Sharpe multipliers now shifted 1 day and applied to position size in a second simulation pass (contracts & costs reflect real size). Ex-ante sizing; no same-day information
4 S1C data snooping QUANTIFIED — direction was chosen after observing S1 fail on 2018–2025; cannot be fixed in code. Tested on 2013–2017, a window never used in the discovery. Pseudo-OOS: S1C +$785K, S1 −$1.31M — direction holds
5 S3 arbitrary P&L scale QUANTIFIED — the 2%/z-unit proxy scale is assumed (no single-stock options data). Sensitivity run at 0.5%/1%/2%/4% per z-unit. P&L +$5K…+$53K, win rate 89–94% at every scale
Honest read: S1C's edge is plausible (survives one falsification attempt) but unproven — confirmation requires live data after 2026-06. Portfolio Sharpe remains below the 5% risk-free hurdle; the system's value is concentrated in crisis convexity, not steady carry.
